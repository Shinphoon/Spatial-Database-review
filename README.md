第3章 数据模型与空间信息模型
§3.1数据模型
E-R模型
组成要素：①实体、②属性、③实体之间的联系。
考试要求：学会绘画E-R图。
关系模型
定义：实体和联系均用二维表(关系模型的数据结构)表示的数据模型。
基本概念：
①关系：表。
②关系模式：表最上面那一行。
③元组：每一行称为关系的一个元组。
④属性：每一列称为关系的一个属性。
⑤域：每一属性所对应的取值范围。
⑥分量：一行里的一个属性值。
⑦主键：表中能唯一标识一行数据的。（唯一、不能为空、每行都必须有）
⑧外键：存在于该表中但不是该表的主键，而是另一个表的主键。（表示“从属关系”、能保证数据之间的关系不会乱掉）
关系模型完整性的三大约束：①实体完整性、②参照完整性、③用户定义完整性。

第4章 关系数据库模型
§4.1 关系模型概述
关系数据模型的常用操作：选择(Select)、投影(Project)、连接(Join)、除(Divide)、并(Union)、交(Intersection)、差(Difference)、查询(Query)、加(Insert)、删除(Delete)、修改(Update)等等。
§4.2 关系数据结构的三大组成部分
(1)域：相同数据类型的值的集合。
(2)笛卡尔积：把两个表中的每一行两两组合一次形成都有可能的搭配。
(3)关系：上面说了，一张表就是一个关系。
【关系中涉及的相关概念】
①属性：列的名字。
②候选码：能够唯一地标识一个元组（行）的属性组。
③主码：当候选码不止一个时，选定其中一个作为主码。
④主属性：主码的属性。
⑤非码属性：不包含在任何候选码中的属性。
⑥全码：关系模式的所有属性组是这个关系模式的候选码。
⑦外码：在一张表里，用来引用另一张表的主码的字段。
§4.3 关系的完整性★
①实体完整性规则：所有主属性不能空、不能重复。
②参照完整性规则：该表的外码必须是另一个表主码的有效值或者是空值。
③用户定义完整性规则：用户按照实际的数据库应用系统运行环境要求，针对某一具体关系数据库的约束条件。（比如说年龄必须0-150、性别只能是男或女等等）
§4.4 关系代数★
(1)	传统的集合运算
并：A∪B			把两张表的记录合在一起（去重）
交：A∩B			找出两张表共同的部分
差：A-B				从A里把和B相同的记录删掉，得到的就是A-B。
广义笛卡尔积：A×B	把A的每一行跟B的每一行全部两两配对
(2)	专门的关系运算
选择：σ逻辑表达式(关系)				按条件过滤行。
投影：π列名/列号(关系)				按条件过滤列。
等值连接：(关系1)⨝逻辑表达式(关系2) 	先做笛卡尔积，再根据逻辑表达式筛选。
自然连接：(关系1)⨝(关系2) 		不写条件，同名字段只保留一个。
除法：(关系1)÷(关系2) 			从关系1中找出所有满足关系2中的每个值的实体。
